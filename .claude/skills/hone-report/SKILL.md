description: >
  Generate a comprehensive review report with executive summary,
  question scorecard, findings by dimension, and recommended actions.
  Saves as a persistent markdown file.
user-invokable: true
args:
  - name: target
    description: File path or name of reviewed spec
    required: false

Generate a persistent review report from the current review session.

## Prerequisites

A Hone review must have been run first. If not, say: "No review data to report on. Run `/hone-review` first."

## Input

Accept a spec name or file path to identify the review. If not provided, use the most recent review in conversation.

## Output Location

Save to: `.hone/reports/YYYY-MM-DD-{spec-name}-review.md`

Create the `.hone/reports/` directory if it doesn't exist.

## Report Structure

Generate a markdown file with this structure:

```markdown
# Hone Review: {Spec Title}

> Reviewed {date} · Size {S/M/L/XL} · {N} questions · Verdict: {VERDICT}

## Executive Summary

{2-3 sentences: what was reviewed, what was found, what the verdict means}

## Classification

| Axis | Score | Reason |
|------|-------|--------|
| Blast radius | {n}/4 | {reason} |
| Reversibility | {n}/4 | {reason} |
| Domain risk | {n}/4 | {reason} |
| Novelty | {n}/4 | {reason} |
| **Total** | **{n}/16** | **Size {X}** |

## Dimensions Reviewed

| Dimension | Findings | Severity | Questions |
|-----------|----------|----------|-----------|
| {name} | {n} | {🟥 n 🟠 n 🟡 n 🔵 n} | {n} asked, {m} answered |
| ... | ... | ... | ... |

## Question Scorecard

| Metric | Value |
|--------|-------|
| Total questions | {n} |
| T1 Clarification | {n} |
| T2 Gap | {n} |
| T3 Challenge | {n} |
| T4 Unknown Unknown | {n} |
| T5 Tradeoff | {n} |
| Answered | {n} |
| Skipped | {n} |
| Deferred | {n} |
| **Confidence** | **{HIGH/MEDIUM/LOW}** |

## Findings

### {Dimension Name}

#### 🟠 {Finding title}

**Location**: {where in spec}
**Question**: {the question that surfaced it}
**Finding**: {what was found}
**Developer response**: {what they chose}
**Action**: {what was done — added to spec, documented as non-goal, etc.}

{repeat for each finding}

## Key Learnings

1. {learning}
2. {learning}
3. {learning}

## Anti-Patterns Detected

{list any anti-patterns found, or "None detected"}

## Accepted Risks

| Risk | Severity | Decision | Rationale |
|------|----------|----------|-----------|
| {risk} | {🟥/🟠/🟡/🔵} | {accepted/deferred} | {why} |

## Recommended Next Steps

1. {action — e.g., "Run /hone-sharpen to apply 4 repairs"}
2. {action — e.g., "Complete research spike on Stripe webhook events"}
3. {action — e.g., "Add webhook signature verification before production"}

## Verdict

**{VERDICT EMOJI} {VERDICT}**

{1-2 sentence verdict explanation}

---

*Generated by Hone 磨く · {N} questions survived · Confidence: {LEVEL}*
```
